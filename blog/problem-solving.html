Was blocked on a simple problem and elegantly solved it

I was stuck on a problem on a coding exercise where I was calling an object, before declaring that object. It’s kind of like reading a story and then all of a sudden some random character (object), out of the blue, was a part of the story the whole time. At first you're confused, but then you read a little more and then you start to understand that random character (object).

I wasn’t understanding why the code couldn’t just keep on reading after calling the object, it would have found the whole explanation of the object. Like duhh? It’s righttt there?. Thinking like this was getting me nowhere, so I had to think really hard on solving the problem.

First thing I did was take a nap, thinking about thinking hard made me not want to do that at the time. So I legit slept on it, and I am glad I did that. Because when I came to solving the problem, I had a fresh mind and perspective to tackle the problem.

So I reread the exercise and code, followed through with my own steps so I could clarify my thoughts at the time when answering. That was it, from there I realized that it was the way I answered. I was looking at my own code as a test marker, thinking ‘why would you answer it like this?’. I realized I had answered multiple questions as a whole and not individually. Which made me trip myself up, by accessing an object before declaring it in that answer.

So instead of answering it as a whole, I answered each individual question. So if I myself viewed it as a marker, I could easily view it and in turn the code became readable.

Throughout the process it was a little tedious but fun at the same time. It was a fun challenge to see how a programme reads what you make and then tells you ‘you make no sense’ with an error message.

This little problem helped me understand even the simplest little bugs can happen to anyone no matter how good you think you are at coding. Making sure you have solid problem solving skills not only in coding, can really help get through a problem and maybe find a better solution. Having experienced this problem will help me later down the line when a similar problem comes about, I would gain experience and effectively use my problem solving techniques to tackle any problem that comes my way.
